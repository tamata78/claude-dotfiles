# Token Audit - トークン消費監査スキル

Claude Codeのトークン消費要因を分析し、具体的な削減施策を提案する。

## 実行手順

### 1. スキャンスクリプトを実行

```bash
python3 ~/.claude/skills/token-audit/scripts/scan.py
```

スクリプトがJSON形式で以下を出力する:
- CLAUDE.mdファイル群のサイズ
- スキル一覧とサイズ（.bakの有無含む）
- プラグインキャッシュ（バージョン重複含む）
- メモリファイルの肥大化
- settings.jsonのサイズ
- 補助ファイル（voicevox-scripts.md等）

### 2. 結果を整形して表示

スキャン結果を以下のフォーマットで表示する:

```
## トークン消費レポート

| カテゴリ | 項目 | サイズ | 推定トークン | 読込タイミング | 備考 |
|----------|------|--------|-------------|---------------|------|
| CLAUDE.md | Global CLAUDE.md | X KB | ~X,XXX tok | 毎ターン | ... |
| スキル | skill-name | X KB | ~X,XXX tok | 呼出時 | ... |
| ... | ... | ... | ... | ... | ... |

合計: ~XX,XXX tokens
```

**読込タイミングの分類**（影響度順）:
- **毎ターン**: CLAUDE.md、スキル説明文 → 最も影響大
- **呼出時**: スキル本体、プラグイン → 使用時のみ
- **ディスクのみ**: キャッシュ重複 → 直接のトークン消費なし（ただし管理上の問題）

### 3. 節約施策を提案

問題を検出したら、投資番号付きで施策を提示する:

```
### 推奨される節約施策

| # | 施策 | 推定削減 | 難易度 | 説明 |
|---|------|----------|--------|------|
| A | ... | ~X,XXX tok | 低 | ... |
| B | ... | ~X,XXX tok | 中 | ... |

どの施策を実行しますか？番号（A〜X）で指定してください。複数選択もOKです。
```

### 4. 施策の実行

ユーザーが番号を選んだら、該当する施策を実行する。
- ファイル削除が必要な場合は、削除対象を明示してユーザーの承認を得てから実行する
- settings.jsonの変更は直接編集で対応する
- CLAUDE.mdの圧縮は変更内容をdiffで見せてから適用する

## 検出ルール

以下の条件に該当する場合、施策として提案する:

| 条件 | 施策カテゴリ |
|------|-------------|
| `.bak` ファイル/ディレクトリが存在する | 削除（git管理で代替） |
| プラグインキャッシュが3バージョン以上 | 古いバージョンを削除 |
| CLAUDE.mdが5,000バイト超 | 内容の圧縮・整理を提案 |
| 使用していないスキルがある | 削除を提案 |
| メモリファイルが20個超 or 合計10KB超 | 古いメモリの整理を提案 |
| settings.jsonの無効化済みプラグイン参照 | 設定の除去 |

## 週次リマインダー

このスキルは毎週日曜日に定期実行の確認をユーザーに投げる設定を推奨する。
初回実行時に、scheduleが未設定であれば以下を案内する:

> 毎週日曜日にトークン監査のリマインダーを設定しますか？
> `/schedule` でCronジョブを作成できます。

設定コマンド例:
```
/schedule create --name "token-audit-weekly" --cron "0 10 * * 0" --prompt "/token-audit"
```
