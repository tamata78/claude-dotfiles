# config-audit - 設定監査スキル

~/.claude/ 配下のトークン消費とルール品質を一括監査し、改善施策を提案する。

## 実行手順

### 1. トークン消費スキャン

```bash
python3 ~/.claude/skills/config-audit/scripts/scan.py
```

以下を分析:
- CLAUDE.md サイズ（毎ターン読み込み）
- スキル一覧とサイズ（.bak含む）
- プラグインキャッシュ（バージョン重複）
- メモリファイルの肥大化
- settings.json サイズ
- 補助ファイル（voicevox-scripts.md等）

### 2. ルール・スキル品質チェック

以下を読み取り:
1. `~/.claude/CLAUDE.md`
2. `~/.claude/commands/` 配下の全 `.md`
3. 現在のプロジェクトの `CLAUDE.md`（存在する場合）
4. `.claude/commands/` 配下（存在する場合）

検出する問題:
- **重複**: 同じ内容が複数箇所に存在
- **矛盾**: 互いに矛盾するルール
- **肥大化**: 1ファイルに詰め込みすぎ（目安50行超）
- **未使用スキル**: 直近で呼ばれていない `~/.claude/skills/*.jsonl` 参照
- **曖昧な記述**: 解釈が分かれそうなルール

### 3. 統合レポートを表示

```
## 設定監査レポート

### トークン消費

| カテゴリ | 項目 | サイズ | 推定トークン | 読込タイミング |
|----------|------|--------|-------------|---------------|
| CLAUDE.md | Global | X KB | ~X,XXX tok | 毎ターン |
| スキル | skill-name | X KB | ~X,XXX tok | 呼出時 |

合計: ~XX,XXX tokens

### ルール・スキル品質

| # | 問題 | アクション | 対象 | 削減/改善 |
|---|------|-----------|------|-----------|
| 1 | ... | 削除/統合/書き換え | path | ~X tokens |
```

### 4. 施策を実行

ユーザーが番号（1, 2, 3...）で選択→実行する:
- ファイル削除: 対象を明示して確認を得てから実行
- settings.json 変更: 直接編集
- CLAUDE.md 圧縮: diff を見せてから適用

## 検出ルール

| 条件 | 提案 |
|------|------|
| `.bak` ファイル/ディレクトリが存在 | 削除（git管理で代替） |
| プラグインキャッシュが3バージョン以上 | 古いバージョンを削除 |
| CLAUDE.md が 5,000 バイト超 | 圧縮・整理を提案 |
| 使用していないスキルがある | 削除を提案 |
| メモリファイルが20個超 or 合計10KB超 | 古いメモリの整理を提案 |
| 同一内容のスキルが複数存在 | 統合または削除 |
| settings.json の hook matcher が `.*` | スコープ絞り込みを提案 |
