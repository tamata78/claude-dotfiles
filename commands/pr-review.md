# /pr-review - PRレビューワークフロー

`$ARGUMENTS` にPR番号を渡す（省略時はリスト表示して選択）。

## 手順

### 1. PRの情報取得

```bash
gh pr list                           # PR一覧（引数なし時）
gh pr view {PR番号}                  # PR詳細
gh pr diff {PR番号}                  # 差分
```

### 2. コードレビュー実行

`/quick-review` と同様の観点でPR差分をレビュー:

#### セキュリティ観点
- ハードコードされた秘密鍵・パスワード・APIキー
- インジェクション脆弱性
- 認証・認可の不備

#### 品質観点
- エラーハンドリングの漏れ
- 型安全性
- テストカバレッジ

#### パフォーマンス観点
- N+1クエリ
- 不必要な処理

### 3. agent-teams のcode-reviewerに委任（大きなPRの場合）

差分が200行を超える場合、`claude-team` のcode-reviewerロールを活用することを提案:
```bash
claude-team code-reviewer
```

### 4. レビューコメントの作成

レビュー結果を構造化してまとめる:

```
## PR #{番号} レビュー: {タイトル}

### 総評
{全体的な評価}

### 🔴 必須対応
- {問題点}: {ファイル:行番号}

### 🟡 推奨対応
- {改善点}: {ファイル:行番号}

### 💡 提案
- {提案内容}

### ✅ 良い点
- {良かった点}
```

### 5. GitHubへのコメント投稿（確認後）

ユーザーに確認してから:
```bash
gh pr review {PR番号} --comment --body "{レビューコメント}"
# または承認
gh pr review {PR番号} --approve --body "LGTM"
# または変更要求
gh pr review {PR番号} --request-changes --body "{コメント}"
```

### 6. 完了通知

```bash
vv "レビュー完了なのだ！"
```
