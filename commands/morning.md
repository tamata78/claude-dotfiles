# /morning - 朝のルーティンワークフロー

朝のセッション開始時に実行。全プロジェクトの状態を把握して作業計画を立てる。

## 手順

### 1. 前回セッションの引き継ぎ

`~/.claude/context/last-session.md` が存在する場合、内容を確認して前回の作業を要約する。

### 2. アクティブプロジェクトのgit状態確認

以下のプロジェクトを確認する（存在するものだけ）:

```bash
for dir in $(find ~/work -maxdepth 1 -mindepth 1 -type d | sort); do
  if [ -d "$dir/.git" ]; then
    echo "=== $(basename $dir) ==="
    cd "$dir" && git status --short && git log --oneline -1
  fi
done
```

未コミット変更があるプロジェクトを一覧表示する。

### 3. 前回のスナップショット確認

各プロジェクトの `.claude/snapshot.md` を確認し、前回どこまで作業したかを要約する。

### 4. 今日の作業提案

以下を踏まえた作業計画を提案する:
- 未コミット変更があるプロジェクト（最優先）
- スナップショットの「次のステップ」
- 前回セッションの続き

提案フォーマット:
```
## 今日の作業候補

1. [プロジェクト名] - [理由]
2. [プロジェクト名] - [理由]

どれから始めますか？または他にやることがあれば教えてください。
```

### 5. 完了通知

```bash
vv "おはようなのだ！今日も頑張るのだ"
```
