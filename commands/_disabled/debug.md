# /debug - デバッグ支援ワークフロー

`$ARGUMENTS` にエラーメッセージまたは問題の説明を渡す。

## 手順

### 1. 問題の整理

`$ARGUMENTS` を解析して問題を構造化する:
- **エラーメッセージ**: そのまま使用
- **症状の説明**: 何が起きているかを整理

### 2. 関連コードの調査

エラーメッセージ内のキーワードでコードベースを検索:

```
Grep で関連ファイルを特定 → Read で詳細確認
```

スタックトレースがある場合は該当行を確認する。

### 3. WebSearch で事例調査

```
WebSearch: "{エラーメッセージの核心部分}" {関連技術スタック}
```

既知の問題・解決策・GitHubイシューなどを収集する。

### 4. 原因の仮説立案

収集した情報から原因候補を列挙:

```
## デバッグレポート

### 現象
{問題の説明}

### 調査結果
- コードベース: {関連箇所の要約}
- 事例調査: {WebSearch結果の要約}

### 原因の仮説（可能性の高い順）
1. {仮説1} - 根拠: {根拠}
2. {仮説2} - 根拠: {根拠}

### 推奨する対処法
{最も可能性の高い修正方法}
```

### 5. 修正の実施

ユーザーが「修正して」と言った場合、最有力仮説に基づいて修正を実施する。

修正後は必ずビルド/テストで動作確認を行う。

### 6. 完了通知

```bash
vv "デバッグ完了なのだ！確認してほしいのだ"
```
