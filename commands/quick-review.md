# /quick-review - クイックコードレビュー

`$ARGUMENTS` にファイルパスやPR番号を指定できる。省略時はgit staged差分をレビュー。

## 手順

### 1. レビュー対象の取得

```bash
git diff --staged    # ステージング済みの変更
# または
git diff HEAD       # 直近の変更全体
```

`$ARGUMENTS` にファイルパスが指定された場合はそのファイルを対象にする。

### 2. レビュー実行

以下の観点でレビューし、問題を検出する:

#### セキュリティ観点
- ハードコードされた秘密鍵・パスワード・APIキー
- SQLインジェクション・XSS・コマンドインジェクションの可能性
- 認証・認可の不備
- 入力値検証の欠如

#### 品質観点
- エラーハンドリングの漏れ
- 非同期処理の適切な待機（async/await漏れ）
- 型安全性（TypeScriptの場合）
- 重複コード・単純化できる処理

#### パフォーマンス観点
- N+1クエリの可能性
- 不必要なループ・重複処理
- メモリリーク要因
- キャッシュ活用の余地

### 3. レビュー結果の報告

以下のフォーマットで報告:

```
## コードレビュー結果

### 🔴 Critical（必ず修正）
- [問題の説明] (ファイル:行番号)

### 🟡 Warning（修正推奨）
- [問題の説明] (ファイル:行番号)

### 💡 Suggestion（改善提案）
- [提案内容] (ファイル:行番号)

### ✅ 問題なし
特に問題は見つかりませんでした。
```

Criticalがある場合は「修正しますか？」と確認し、承認されれば修正まで実施する。

### 4. 完了通知

```bash
vv "レビュー完了なのだ！"
```
