# dependency-audit スキル

依存パッケージの陳腐化を検出し、安全な更新計画を立案する。`$ARGUMENTS` にプロジェクトパスを指定できる（省略時はCWD）。

## 実行手順

### 1. 技術スタックの確認

- `package.json` の有無 → Node.js プロジェクト
- `requirements.txt` / `pyproject.toml` の有無 → Python プロジェクト

### 2. 更新可能パッケージのスキャン

**Node.js:**
```bash
npm outdated 2>/dev/null
```

**Python:**
```bash
pip list --outdated 2>/dev/null
```

### 3. 破壊的変更リスクの評価

各パッケージについて:
- **メジャーバージョンアップ** → 破壊的変更の可能性あり（要確認）
- **マイナーバージョンアップ** → 低リスク（新機能追加）
- **パッチバージョンアップ** → 最低リスク（バグ修正のみ）

重要パッケージ（フレームワーク・コアライブラリ）のCHANGELOGをWebSearchで確認:
```
WebSearch: "{パッケージ名} v{旧バージョン} to v{新バージョン} breaking changes migration"
```

### 4. 更新計画の生成

```
## 依存関係監査レポート - {プロジェクト名}

| パッケージ | 現在 | 最新 | リスク | 推奨アクション |
|-----------|------|------|--------|---------------|
| {パッケージ} | {旧} | {新} | 低/中/高 | 即時更新/確認後更新/保留 |

### 即時更新推奨（パッチ・マイナー）
```bash
npm update {パッケージ1} {パッケージ2}
```

### 確認後に更新（メジャー）
- {パッケージ}: {確認すべき変更点のサマリー}

### 保留（互換性リスクあり）
- {パッケージ}: {理由}
```

### 5. セキュリティパッチの優先対応

`npm audit` または `pip-audit` と組み合わせて、セキュリティパッチが含まれる更新を最優先で提案する。

### 6. 更新実行

ユーザーが承認した場合:
- 低リスクパッケージは一括更新
- 高リスクは1件ずつビルド/テスト確認しながら更新

```bash
vv "依存関係の監査完了なのだ！"
```
